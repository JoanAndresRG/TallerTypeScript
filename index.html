Taller TypeScript

¿Qué es TypeScript?

Es un lenguaje de programacion fuertemente tipado de código abierto, desarrollado y mantenido por Microsoft. TypeScript extiende la sintaxis de JavaScript, por tanto cualquier código JavaScript existente debería funcionar sin problemas.

Está pensado para grandes proyectos, los cuales a través de un compilador de TypeScript se traducen a código JavaScript original.

TypeScript es usado para desarrollar aplicaciones JavaScript que se ejecutarán en el lado del cliente o del servidor, o extensiones para programas

¿Que son los tipos de datos?

El sistema de tipos de Typescript realiza una formalización de los tipos de Javascript, mediante una representación estática de sus tipos dinámicos. Esto permite a los desarrolladores definir variables y funciones tipadas sin perder la esencia de Javascript. Poder definir los tipos durante el tiempo de diseño nos ayuda a evitar errores en tiempo de ejecución, como podría ser pasar el tipo de variable incorrecto a una función.

TypeScript, como JavaScript, le permite trabajar con matrices de valores. Los tipos de matriz se pueden escribir de dos formas. En el primero, usa el tipo de los elementos seguido de []para denotar una matriz de ese tipo de elemento: let list: number[] = [1, 2, 3];

Los tipos de tupla le permiten expresar una matriz con un número fijo de elementos cuyos tipos son conocidos, pero no es necesario que sean los mismos. Por ejemplo, es posible que desee representar un valor como un par de string y number: // Declare a tuple type let x: [string, number]; // Initialize it x = ["hello", 10]; // OK // Initialize it incorrectly x = [10, "hello"]; // Error Type 'number' is not assignable to type 'string'. voides un poco lo contrario de any: la ausencia de tener ningún tipo en absoluto. Comúnmente, puede ver esto como el tipo de retorno de funciones que no devuelven un valor: function warnUser(): void { console.log("This is my warning message"); } Declarar variables de tipo voidno es útil porque solo puede asignar null(solo si strictNullChecksno se especifica, consulte la siguiente sección) o undefined ellas: let unusable: void = undefined; // OK if `--strictNullChecks` is not given unusable = null;
